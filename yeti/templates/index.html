YETI

YETI is an implementation of the Trusted Automated eXchange of Indicator Information (TAXII).

Overview

YETI is an implementation of TAXII. YETI was written in Python 2.7 and utilizes the Django 1.5 web application framework. YETI implements TAXII Services as defined by the TAXII Services Specification and the TAXII HTTP Binding Specification.

Services

The following TAXII services are implemented by YETI. The services listed below are defined by the TAXII Services Specification.

Discovery

The Discovery service is the mechanism for communicating information related to the availability and use of TAXII Services. The Discovery service can be accessed by transmitting a Discovery request message to the /services/discovery/ location of this YETI installation. YETI comes bundled with a discovery_client.py script to serve as a demonstration of this service. The discovery_client script.py script is located in the scripts directory of your YETI installation.

Inbox

The Inbox service is the mechanism by which a TAXII consumer accepts messages from a TAXII producer in producer-initiated exchanges (i.e., push messaging). The Inbox service can be accessed by transmitting an Inbox message to the /services/inbox/<inbox_name> location of this YETI installation. For example, sending data to the default inbox established by a quickstart YETI instance would require sending an Inbox message to http://localhost:8080/services/inbox/default/. YETI comes bundled with a inbox_client.py script to serve as a demonstration of the Inbox service. This script is located in the scripts directory of your YETI installation.

Poll

The Poll service is the mechanism by which a TAXII consumers can pull data from a TAXII Data Collection. The Poll service can be accessed by transmitting a Poll request to the /services/poll/ location of this YETI installation. The name of the TAXII Data Collection for the Poll request is contained within the TAXII message itself and not specified within the URL. To Poll data from the default Data Collection established by a quickstart YETI instance, a TAXII client would transmit a Poll request to http://localhost/services/poll/. YETI comes bundled with a poll_client.py script to serve as a demonstration of the Poll service. This script is located in the scripts directory of your YETI installation.

Administration

YETI includes an administrator interface that supports the ability to add user accounts, create and modify Data Collections and Inboxes, see and edit stored content, among other capabilities. To access the administrator interface, navigate to /admin and enter your superuser credentials (these are created when YETI is first run).
